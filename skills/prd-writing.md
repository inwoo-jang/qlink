# PRD 작성 스킬

## 출력 템플릿
```
1. 제품 한 줄 요약
2. 문제 정의 (Pain Point)
3. 타겟 사용자 / 페르소나 (1~3개)
4. 사용 시나리오 (Top 3)
5. 핵심 가치 제안 (Value Prop)
6. 핵심 기능 (MVP / v1.x / Backlog)
7. 비기능 요구사항 (성능·보안·접근성·다국어)
8. 성공 지표 (KPI)
9. 리스크 / 가정
10. 마일스톤
```

## 큐링크 적용 예시 (요약)
- **MVP**: URL 저장, QR 스캔, AI 요약(폴백 API), 폴더, 검색, 공유
- **v1.1**: 알림, 태그 자동분류 고도화, 다국어 요약
- **v1.2**: 협업 폴더, 브라우저 익스텐션
- **KPI**: 주간 저장 링크 수, 7일 재방문율, 요약 만족 클릭율

## 체크리스트
- [ ] 페르소나가 사용자 페인을 구체적으로 묘사하는가?
- [ ] 각 기능에 인수 기준(Acceptance Criteria)이 있는가?
- [ ] 비기능 요구사항이 측정 가능한가?
- [ ] 우선순위가 명시되어 있는가?
