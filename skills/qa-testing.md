# QA 테스트 스킬

## 테스트 피라미드
```
        ▲ E2E (Playwright)
       ███ 통합 테스트
      █████ 컴포넌트 (Vitest + RTL)
     ███████ 단위 (Vitest)
```

## 테스트 케이스 템플릿
```
TC-ID: TC-LINK-001
제목: 클립보드의 URL 자동 인식 후 저장
선행조건: 클립보드에 https://... URL 복사됨
스텝:
  1. 홈에서 (+) 버튼 탭
  2. 시트 진입 시 URL 자동 채워짐 확인
  3. "저장" 탭
기대결과:
  - 홈 피드 최상단에 새 카드 노출
  - 토스트 "저장됨" 노출
  - 5초 이내 AI 요약 표시
디바이스: iPhone 14 (Safari), Pixel 7 (Chrome)
우선순위: P0
```

## 우선순위 정의
- P0: MVP 차단 (저장/검색/AI요약/QR)
- P1: 주요 기능 (폴더링/공유/알림)
- P2: 보조 기능 (테마/다국어 외)

## 핵심 회귀 시나리오 (E2E)
1. URL 저장 → 카드 노출 → 상세 진입 → 외부 열기
2. QR 스캔(모킹된 카메라) → 저장
3. 검색어 입력 → 필터링 → 결과 클릭
4. 폴더 생성 → 카드 이동 → 폴더 상세
5. 알림 등록 → 푸시 수신(테스트 푸시)
6. PWA 설치 → 오프라인 모드에서 캐시된 카드 노출

## 모바일 디바이스 매트릭스
| 디바이스 | 브라우저 | 우선도 |
|---------|---------|-------|
| iPhone (iOS 17+) | Safari | P0 |
| Android 13+ | Chrome | P0 |
| iPad | Safari | P1 |
| Galaxy | Samsung Internet | P2 |

## 자동화 도구
- Playwright(크로스브라우저), `@playwright/test`
- `npx playwright codegen` 으로 시작 → 안정 셀렉터로 다듬기
- CI에서 main 머지 시 실행
