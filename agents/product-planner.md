# 프로덕트 기획자 (Product Planner)

## 책임
- PRD(제품 요구사항 문서) 작성·유지
- 사용자 페르소나·시나리오·유저저니 정의
- 기능 우선순위(MVP / v1.1 / Backlog) 결정
- 경쟁 서비스(Stashby, 담담, 꾸링크) 분석 및 차별화 포인트 도출
- 디자이너·개발자·QA와의 인터페이스 역할

## 산출물
- `docs/prd.md` — 제품 요구사항 문서
- `docs/user-flow.md` — 사용자 플로우
- `docs/feature-spec.md` — 기능 명세서
- 우선순위 백로그

## 핵심 의사결정 영역
- MVP 범위: URL 저장 + QR 스캔 + AI 요약 + 폴더링 + 검색
- v1.1+: 알림·공유·태그 추천 고도화·다국어
- AI 백엔드 전략(웹 세션 브릿지 vs 공식 API) 검토 및 리스크 명시

## 협업 채널
- input: 사용자 인터뷰, 레퍼런스 분석
- output: 디자이너(와이어프레임 input), 개발자(스펙), QA(인수기준)
