# 큐링크 (QLINK) PRD

작성일: 2026-05-07
버전: v1.1
작성자: 인우

## 1. 개요

링크를 **저장하고 끝**이 아니라 **할 일과 반복 알림으로 변환해 행동까지 이끄는** 모바일·웹 큐레이션 북마크 서비스.

한 링크에 여러 개 할 일을 매달 수 있고, 1회 알림과 매일·평일·매주·매월 반복 알림 모두 지원. 출퇴근 모바일과 회사·집 데스크톱이 한 계정으로 끊김 없이 동기화. 정보 수집형 직장인의 회의 D-day, 취준생의 채용 마감일, 자격증 학습자의 매일 진도 등 시간 압박 시나리오에 직접 대응.

상위 문서: `01_프로젝트개요.md`

## 2. 타겟 유저 페르소나 + 사용 시나리오

> 페르소나는 사용 시나리오와 짝지어 읽을 때 의미가 있다. 본 절은 시나리오 → 페르소나 → 핵심 페인 → 기대 가치 순으로 정리.

### 2.1 주 타겟 — 정보 수집형 직장인

**페르소나**

| 항목 | 내용 |
|---|---|
| 이름/나이/직업 | **김지훈, 30, IT 회사 PM** (또는 마케터·기획자·개발자) |
| 사용 디바이스 | iPhone 13 + 회사 MacBook + 집 LG 그램 (3개) |
| 일일 링크 노출 | 30~50개 (슬랙·노션·트위터·링크드인·뉴스레터·단톡방·유튜브) |
| 사용 시간대 | 출퇴근 지하철 모바일(아침 30분/저녁 40분), 회사 데스크톱 점심·자투리, 주말 집 PC |

**대표 시나리오 — 월요일 아침**

> 9시. 회의 시작 전 10분. 슬랙에서 팀장이 노션 링크 5개를 보냈다. "이번 주에 다 검토해주세요." 카톡에서 친구가 보낸 신기한 글, 트위터에서 본 회사 케이스 스터디까지 더해 모은 링크가 8개. **평소 같으면 별표 누르고 끝**. 그러나 QLINK 이후:
>
> 1. 슬랙에서 노션 링크 → "공유 → QLINK" → 자동으로 #LINK-ADD에 진입
> 2. AI가 1.4초 안에 한 줄 요약 ("Q3 마케팅 전략 — KPI·예산·실행 일정")
> 3. **할 일 3개를 한꺼번에 추가**:
>    - "수요일 9시 회의 전 1차 검토" — 1회 알림 (수 8:30)
>    - "내용 요약 메모 작성" — 1회 알림 (목 18:00)
>    - "다음 회의에 가져갈 의견 정리" — 1회 알림 (금 14:00)
> 4. 폴더 자동 추천: "팀 회의록"
> 5. 8개 링크 처리 끝 — 5분 안에 완료
>
> **3시간 뒤 회사 데스크톱**에서 같은 노션 링크를 펼치면 같은 할 일 3개가 그대로. 하나씩 체크해 나간다. **수요일 8:30**에 알림이 오면 한 번 더 보고 회의에 들어간다.

**핵심 페인포인트** (왜 QLINK가 필요한가)
- "캡처만 하고 결국 안 본다 — 행동으로 옮길 트리거가 없다"
- "한 링크에 여러 액션이 붙는데 메모 한 줄로는 부족하다"
- "회사 PC와 집 PC가 따로 놀아서 결국 어딘가에서 잃어버린다"
- "단톡방 무한 스크롤로 친구가 보낸 그 링크 다시 못 찾는다"

**기대 가치**
- 1초 저장 (공유 시트 → 자동 채움)
- AI 한 줄 요약으로 진짜 읽을 가치 즉시 판단
- **"링크 = 작업 단위" 사고방식 — 일이 줄어드는 느낌**
- 모든 디바이스 자연 동기화

### 2.2 부 타겟 1 — 취준생 / 채용 지원자

**페르소나**

| 항목 | 내용 |
|---|---|
| 이름/나이/직업 | **박서연, 26, 취업 준비 6개월 차** (마케팅·기획 직군) |
| 일일 링크 노출 | 사람인·잡코리아·원티드·링커리어 + 합격수기 블로그 = 주 10~15 공고 + 멘토 글 |
| 가장 큰 후회 | **"마감일 놓쳐서 지원조차 못 한 회사가 3~4곳"** |
| 사용 시간대 | 카페·도서관 모바일 / 집 데스크톱 |

**대표 시나리오 — 채용 공고 발견 → 마감일 전 자동 트래킹**

> 잡코리아에서 마음에 드는 공고 발견. 마감일은 5월 14일.
>
> 1. 공고 페이지에서 "공유 → QLINK" → 자동 저장
> 2. AI 요약: "데이터 마케팅 인턴 — 5/14 마감, 주 4일, 강남"
> 3. 폴더 "지원 진행 중"으로 자동 분류
> 4. **할 일 4개 일괄 등록**:
>    - "자기소개서 초안 작성" — 5/11 09:00 알림 (마감 D-3)
>    - "포트폴리오 PDF 첨부 확인" — 5/13 09:00 알림 (마감 D-1)
>    - "지원 완료" — 5/14 23:59 알림 (당일)
>    - 면접 잡히면 → "면접 D-day 매일 9시 복습 (반복: 매일, 면접 일자까지)"
> 5. 폴더 보기에서 마감일 순 자동 정렬
> 6. 할일 탭 "🔥 기간지남" 필터로 놓친 게 없는지 한눈에
>
> **반년이 지나도 한 건도 마감일을 놓치지 않는다.** 친구·취준 동기와 "지원 진행 중" 공유 폴더 만들어 서로 응원과 정보 교환.

**핵심 페인**
- "공고 마감일 놓치는 게 가장 무서움"
- "면접 D-day까지 매일 정해진 시간 복습 루틴이 필요한데 캘린더와 따로"
- "친구·동기와 정보 교환할 때 단톡방에 매번 같은 링크 다시 보냄"

**기대 가치**
- 마감일 자동 트래킹으로 **놓침 0건**
- 면접 준비 매일 반복 알림
- 동기와 공유 폴더로 정보 큐레이션

### 2.3 부 타겟 2 — 시험·자격증·IT 학습자

**페르소나**

| 항목 | 내용 |
|---|---|
| 이름/나이/직업 | **이민호, 28, 정보처리기사 + SQLD 동시 준비 + IT 부트캠프 수강** |
| 일일 링크 | 인프런 강의·유튜브 강의·기출 PDF·합격수기 블로그 (50개+) |
| 사용 패턴 | 매일 같은 시간 진도 빼야 하지만 캘린더와 북마크가 따로 |

**대표 시나리오 — 강의 + 매일 반복 알림**

> 인프런에서 "정보처리기사 알고리즘" 강의 결제. 35편짜리, 시험까지 2개월.
>
> 1. 강의 메인 페이지를 QLINK에 저장
> 2. **반복 할 일 등록**:
>    - "오늘 1편 듣기" — 반복: 평일 21:00, 시험 1주일 전까지 자동 종료
>    - "매주 토요일 기출 풀이" — 반복: 매주 토요일 10:00
>    - "오답 노트 정리" — 1회, 시험 1주일 전 (5/30)
> 3. 강의 링크 하나에서 **약 30번의 알림**이 자동 생성
> 4. 매일 21:00 푸시 → 한 클릭에 강의 페이지로
> 5. 들은 회차는 할 일 체크 → 진도율 시각화 (TBD: 진도 위젯)

**핵심 페인**
- "캘린더 따로, 북마크 따로 — 매일 옮겨 적기 귀찮음"
- "한 강의 링크에 '강의 듣기 / 기출 풀기 / 노트 정리' 같이 여러 작업 필요"

**기대 가치**
- 강의 = 1개 링크 = N개 할 일 + 반복 알림
- 진도 트래킹 + 알림 ON/OFF로 학습 페이스 자체 조절
