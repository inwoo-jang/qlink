# 큐링크 (QLINK) PRD

작성일: 2026-05-07
버전: v1.0
작성자: 인우

## 1. 개요

URL/QR로 모은 링크를 AI가 한 줄 요약하고 폴더·태그로 분류해주는, **모바일·웹을 한 계정으로 잇는** 큐레이션 북마크 서비스. 정보 수집형 직장인이 출퇴근 지하철 모바일에서 빠르게 정리하고, 회사·집 PC에서 끊김 없이 이어 본다. 단톡방에 묻힌 링크와 통제 잃은 크롬 즐겨찾기 문제를 동시에 해결.

상위 문서: `01_프로젝트개요.md`

| 항목 | 내용 |

| 이름/나이/직업 | 김지훈 / 30 / IT 회사 마케터 (또는 PM/기획자/개발자) |
| 사용 맥락 | 출퇴근 지하철·점심시간 **모바일**에서 트위터·링크드인·뉴스레터·단톡방·인스타·유튜브 링크를 빠르게 저장. 회사 데스크에서 **웹**으로 펼쳐서 진짜 읽을 것을 추리고, 집 PC에서 한 번 더 정리. 한 계정으로 모든 디바이스 동기화 |
| 주요 페인포인트 | "인스타였는지 네이버였는지 헷갈려서 못 찾음" / "단톡방에 묻힌 링크 무한 스크롤" / "유튜브 영상 저장해도 결국 다시 안 봄" / "크롬 즐겨찾기 폴더가 통제 없이 늘어나서 못 찾음" / "회사·집 크롬 계정이 달라 즐겨찾기가 사라짐" |
| 기대 가치 | 저장 즉시 한 줄 요약 → 진짜 읽을 가치 판단 / 키워드로 본문·요약 동시 검색 / 모바일·웹 자동 동기화 / 친구와 공유 폴더로 같이 큐레이션 |

## 5. 비기능 요구사항

| 카테고리 | 요구사항 |
|---|---|
| 성능 | 첫 화면 LCP ≤ 2.5초 (3G Fast). 검색 입력 후 200ms 디바운스 → 즉시 결과. AI 요약 1.4초 이내 응답(폴백 mock 포함). 무한 스크롤 페이지당 30개. **모바일↔웹 동기화 지연 ≤ 2초** |
| 제약 조건 | URL ≤ 2,048자 / 제목 ≤ 200자 / 요약 ≤ 500자 / 태그 ≤ 5개·각 20자 / 폴더명 ≤ 30자 / 폴더 ≤ 100개·유저 / 링크 ≤ 10,000개·유저 (TBD). 크롬 import 파일 ≤ 5MB |
| 호환성 | **모바일**: iOS Safari 15+, Android Chrome 100+ / **데스크톱 웹**: Chrome/Safari/Edge/Firefox 최신 2버전, 1024px·1440px·1920px 반응형 / PWA 설치 모두 지원 |
| 보안 | HTTPS 필수. JWT 액세스 토큰(15분) + 리프레시(30일). 비밀번호 bcrypt. OAuth 2.0/PKCE. CORS 화이트리스트. httpOnly 쿠키 또는 secure storage |
| 데이터 관리 | AWS RDS 암호화 저장(KMS). soft delete + 30일 후 hard delete (TBD). 일 1회 스냅샷, 7일 보관. 사용자 요청 시 다운로드·완전 삭제 제공 |
| 운영체제 권한 | 카메라(QR 스캔), 알림(권장), 클립보드 읽기. 모두 미허용 시 폴백 동작 정의 |
| 오프라인 | 미지원. 마지막 본 화면 PWA 캐시로 표시. 새 저장·검색 네트워크 필요. 오프라인 큐잉은 v2 백로그 |
| 동기화 | 단일 백엔드(AWS RDS) → 디바이스 간 충돌 없음. WebSocket Realtime 또는 5s polling. 동시 편집 시 last-write-wins (TBD: CRDT 검토) |
| 접근성 | WAI-ARIA 라벨, 키보드 네비게이션(데스크톱 단축키 N/J/K/⌘K 등), 색 대비 WCAG AA |
| 다국어 | v1: 한국어만. v1.x: 영어 추가 검토 |
