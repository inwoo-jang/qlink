## 2. 플로우 상세

### FLOW-001: 회원가입 (이메일)

| 항목 | 내용 |
|---|---|
| 시작점 | #AUTH-LOGIN |
| 종료점 | #HOME (자동 로그인) |
| 트리거 | "회원가입" 탭 또는 처음 방문 |

#### Happy Path

| Step | 화면 | 사용자 액션 | 시스템 반응 | 다음 화면 |
|---|---|---|---|---|
| 1 | #AUTH-LOGIN | "회원가입" 탭 | 가입 폼으로 전환 | #AUTH-SIGNUP |
| 2 | #AUTH-SIGNUP | 이메일·비밀번호·닉네임 입력, "가입" | 입력 검증 + signUp 호출 | (대기) |
| 3 | #AUTH-EMAIL-VERIFY | 인증코드 입력 | 코드 검증 + 프로필 자동 생성 | #HOME |

#### 분기 케이스

| 분기 | 조건 | 처리 |
|---|---|---|
| 이메일 형식 오류 | 입력 검증 | 인풋 강조 + "올바른 이메일 형식이 아닙니다" |
| 중복 이메일 | 가입 호출 결과 | "이미 사용 중인 이메일입니다" + 로그인 탭 유도 |
| 비밀번호 정책 미충족 | 입력 검증 | "8자 이상, 영문+숫자 포함" 안내, 가입 버튼 비활성 |
| 인증 코드 불일치 | 3단계 | "인증코드가 일치하지 않아요" + 재시도 |
| 인증 코드 만료 (10분) | 3단계 | "인증 시간이 만료됐어요. 재발송해주세요" + 재발송 |
| 네트워크 오류 | 어느 단계든 | 토스트 "네트워크 연결을 확인해주세요" + 폼 상태 유지 |
| 프로필 자동 생성 실패 | 인증 후 트리거 | 1초 대기 후 1회 재시도 → 실패 시 로그아웃 + 에러 안내 |

### FLOW-009: 모바일↔웹 자동 동기화

| 항목 | 내용 |
|---|---|
| 시작점 | 어느 디바이스든 데이터 변경 발생 |
| 종료점 | 다른 디바이스에 1~2초 내 반영 |
| 트리거 | 링크 INSERT/UPDATE/DELETE, 폴더 변경, 멤버 변경 |

#### Happy Path

| Step | 화면 | 사용자 액션 | 시스템 반응 | 다음 화면 |
|---|---|---|---|---|
| 1 | (모바일) #HOME | 링크 추가 | 서버 INSERT + Realtime publish | — |
| 2 | (웹) #HOME | (자동) | Realtime 구독 → 새 카드 fade-in 삽입 | — |
| 3 | (모바일) #LINK-DETAIL | 폴더 이동 | UPDATE + Realtime publish | — |
| 4 | (웹) #FOLDER-DETAIL | (자동) | 카드가 다른 폴더로 부드럽게 이동 | — |

#### 분기 케이스

| 분기 | 조건 | 처리 |
|---|---|---|
| 동시 편집 (같은 링크 두 디바이스에서 수정) | 충돌 | last-write-wins (`updated_at` 기준). 우선 v1, 추후 CRDT 검토 |
| Realtime 연결 끊김 | 네트워크 | 재연결 시 마지막 sync timestamp 이후 변경분 일괄 fetch |
| 백엔드 publish 실패 | 서버 | 클라이언트는 로컬 적용, 다음 polling(60s)에서 검증 |

### 💻 FLOW-020: Chrome 확장 — 양방향 즐겨찾기 sync 권한 동의

**사용자**: 이미 Chrome 즐겨찾기에 수년치 북마크가 쌓여 있고, QLINK도 사용 중. 양쪽이 자동으로 동기화되길 원함.

#### Happy Path

| Step | 화면 | 사용자 액션 | 시스템 반응 |
|---|---|---|---|
| 1 | #SETTINGS-EXT | "Chrome 즐겨찾기 양방향 sync" 토글 ON | 확장에 `chrome.permissions.request({permissions: ['bookmarks']})` 요청. Chrome이 권한 다이얼로그 표시 |
| 2 | 권한 다이얼로그 | "허용" 클릭 | 확장이 `chrome.bookmarks.getTree()` 호출 → 전체 트리 fetch → 서버에 초기 sync (한 번에 일괄 INSERT) |
| 3 | 서버 | initial sync 완료 | 각 Chrome 폴더 → QLINK 폴더 (이모지 자동 할당), 각 북마크 → QLINK 링크 (요약 미생성 상태) → AI 큐로 enqueue |
| 4 | 백그라운드 | 사용자가 Chrome에서 새 북마크 추가 | `chrome.bookmarks.onCreated` → 확장 → 서버 `POST /links` → 다른 디바이스로 WebSocket 전파 |
| 5 | 백그라운드 | 사용자가 QLINK 웹에서 링크 추가 | `POST /links` 응답 → 확장 service worker가 polling 또는 WebSocket 수신 → `chrome.bookmarks.create({parentId, title, url})` |
| 6 | #SETTINGS-EXT | sync 상태 표시 | "마지막 sync: 방금 전 · 12,847개 항목" + "일시 정지" / "초기화" 버튼 |

#### 분기 케이스

| 케이스 | 처리 |
|---|---|
| 권한 거절 | sync 비활성 유지. newtab override는 영향 없음. 토스트 "권한 없이는 sync 불가. IMPORT-001(HTML 임포트)는 사용 가능" |
| 초기 sync 중 충돌 (양쪽에 동명 폴더) | 폴더명 뒤 ` (Chrome)` 접미사 자동 추가, 사용자 후속 정리 |
| Chrome에서 폴더 삭제 → QLINK에서 미분류로 이동 안 시키고 폴더 자체 삭제? | 정책: **Chrome 삭제 = QLINK 폴더 soft delete (휴지통, 30일 보관)**. 내부 링크는 미분류로 이동 |
| QLINK에서 link 삭제 → Chrome 북마크도 즉시 삭제 | 사용자가 처음 sync ON 할 때 명시 동의("양방향 = 삭제도 동기화됨"). 별도 토글 없음 |
| 일시 정지 ON | 양방향 sync 즉시 중단. 재개 시 정지 기간 중 변경 사항 diff sync |
| 5,000개 초과 북마크 | 초기 sync는 청크(500개씩) + 진행률 바. AI 요약은 lazy(사용자가 카드 열 때 생성) |
| 다중 디바이스 + 확장 다수 | 확장 ID별 device_id, 같은 user에서는 last-write-wins (SYNC-001 정책 재사용) |
| 사용자가 sync OFF 토글 | 권한은 유지, 이벤트 리스너만 detach. 다시 ON 시 권한 재요청 없이 즉시 재개 |
