## B. 기능명세_상세

> 화면 ID는 `#`로 시작 (예: `#HOME`, `#LINK-ADD`). 본 문서의 화면 ID는 추후 IA 문서에서 정식 등록.

### AUTH-001 — 이메일 회원가입
**화면 ID**: `#AUTH-LOGIN`, `#AUTH-SIGNUP`, `#AUTH-EMAIL-VERIFY`

| 항목 | 내용 |
|---|---|
| 설명 | 이메일·비밀번호·닉네임으로 계정 생성. 이메일 인증코드 검증 후 자동 로그인. 진입점 A: #AUTH-LOGIN의 "회원가입" 탭. 진입점 B: 게스트 모드 헤더 "가입하기" 버튼 (FLOW-003) |
| 입력 | - 이메일: 형식 검증 + 중복 불가 / - 비밀번호: 8~64자, 영문 + 숫자 필수, 특수문자 가능 / - 닉네임: 1~20자, 한글/영문/숫자 / - 인증코드: 6자리 숫자 |
| 출력 | - 인증 메일 발송 / - 인증 성공 시 JWT 액세스+리프레시 토큰 발급 / - profiles 자동 생성 후 #HOME |
| 정책 | [기본값] 아바타: 🌸 / [시간] 인증코드 유효 10분 / [재시도] 재발송 시 이전 코드 무효 / [처리 순서] 인증 완료 전까지 가입 진행 불가 / [동기화] 가입 완료 즉시 모든 디바이스 로그인 가능 (POLICY-AUTH 참고) |
| 예외 | - 이메일 형식 오류: '올바른 이메일 형식이 아닙니다' / - 중복 이메일: '이미 사용 중인 이메일입니다' + 로그인 탭 유도 / - 비밀번호 정책 미충족: '8자 이상, 영문+숫자 포함이 필요해요' / - 인증코드 불일치: '인증코드가 일치하지 않아요' / - 인증코드 만료: '인증 시간이 만료됐어요. 재발송해주세요' / - profiles 자동 생성 실패: 1초 대기 후 1회 재시도 → 실패 시 로그아웃 + 안내 / - 네트워크 오류: '네트워크 연결을 확인해주세요' |
| API 연동 | 회원가입 API, 이메일 인증 코드 발송 API, 인증 코드 검증 API, 프로필 생성 트리거 |
| 메모/이슈 | ✅ 결정: 이메일 6자리 코드. SMS는 v1.x 검토 |

### LINK-001 — URL 직접 입력 추가
**화면 ID**: `#HOME`, `#LINK-ADD`

| 항목 | 내용 |
|---|---|
| 설명 | 사용자가 URL을 직접 입력 또는 붙여넣어 저장하는 1차 진입 경로. + 버튼 → 시트 슬라이드 업 → 입력 → 저장. 저장 직후 LINK-004(AI 요약) 자동 트리거 |
| 입력 | - URL (필수, 2048자 이하, http/https) / - 폴더 (선택, 기본=미분류) / - 메모 (선택, 200자) / - (자동) source_type='url', owner_id |
| 출력 | - links INSERT → 홈 피드 최상단에 카드 fade-in / - 토스트 "저장됐어요" / - LINK-004 비동기 트리거 |
| 정책 | [기본값] 폴더=미분류 / [제약] URL 길이 ≤ 2048자, 한 유저당 ≤ 10,000개 / [중복 처리] 동일 owner의 동일 URL은 INSERT 차단 / [동기화] INSERT 직후 SYNC-001로 다른 디바이스에 1~2초 내 반영 |
| 예외 | - URL 형식 오류: '올바른 URL 형식이 아니에요' + 저장 버튼 비활성 / - 중복 URL: 모달 "이미 저장된 링크예요. 상세로 이동할까요?" → "이동" 또는 "취소" / - 한도 초과 (10,000개): '저장 한도에 도달했어요. 오래된 링크를 정리해주세요' / - 네트워크 오류: 로컬 큐잉 + '곧 다시 시도할게요' (POLICY-LOCAL 참고) |
