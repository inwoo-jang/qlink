# 큐링크 (QLINK)

작성일: 2026-05-07
버전: v1.1
작성자: 인우

## 1. 프로젝트명

큐링크 (QLINK)

## 2. 한줄 설명

링크를 **저장하고 끝**이 아니라 **할 일과 반복 알림으로 변환해 행동까지 이끄는** 모바일·웹 큐레이션 북마크.

## 3. 배경 및 목적

### 배경

링크를 받는 채널이 폭증했다. 단톡방·인스타DM·트위터·링크드인·뉴스레터·유튜브·슬랙·노션·잡코리아·사람인 — 하루 30~50개 링크가 흘러들어온다. 사용자는 "나중에 봐야지" 하고 어딘가에 저장하지만, 막상 다시 찾으려 하면 어디서 봤는지조차 헷갈린다. 더 큰 문제는 **저장만 하고 행동으로 옮기지 못한다**는 것 — 채용 공고는 마감일을 넘기고, 강의 영상은 찜해두고 안 보고, 회의록 링크는 회의 직전에야 다시 떠오른다.

기존 북마크 매니저는 "정리"까지만 한다. QLINK는 그 다음 단계인 **"링크 → 할 일 → 알림 → 완료"** 흐름을 한 자리에 묶는다. 동시에 단일 PWA 코드베이스로 모바일·웹을 동시 지원하고, 계정 단위로 끊김 없이 동기화한다.

### 페인포인트

**찾기 어려움 (저장 단계)**
- "인스타에서 봤는지 네이버 카페에서 봤는지 헷갈려서 못 찾는 링크가 너무 많다"
- "단톡방에서 친구가 공유한 링크가 새 메시지로 묻혀서 끝없이 스크롤을 올려야 한다"
- "북마크에 저장해도 제목만 있고, 한 달 뒤엔 뭐였는지 기억이 안 난다"

**행동 단절 (실행 단계) ← QLINK의 핵심 차별점**
- "유튜브 강의를 저장만 해두고 결국 다시 안 본다"
- **"채용 공고를 캡처해뒀는데 마감일을 넘겨서 못 지원한 게 한두 번이 아니다"**
- **"강의 영상에 매일 같은 시간 진도 알림이 있으면 좋겠는데, 캘린더와 북마크가 따로 놀아 매번 옮겨 적기 귀찮다"**
- **"한 회의록 링크에 '검토하기 / 메모 정리 / 다음 회의에서 공유'처럼 여러 액션이 붙는데 메모 한 줄로는 부족하다"**
- "친구들과 같은 주제 링크를 공유할 때 매번 단톡방에 다시 보내야 한다"

**디바이스 단절 (멀티플랫폼)**
- "크롬 즐겨찾기 폴더가 통제 없이 끝도 없이 늘어나서, 정작 필요할 때 못 찾는다"
- "회사 PC와 집 PC, 그리고 휴대폰 — 크롬 계정이 다르거나 동기화가 끊기면 즐겨찾기가 사라지거나 다른 게 나타난다"
- "PC에서 본 링크를 모바일에서 이어서 보고 싶은데 늘 어딘가에 따로 저장된다"

### 사용 시나리오 (3가지 핵심)

#### 시나리오 1 — 직장인 업무 (주 타겟)

> **김지훈, 30세, IT 회사 PM.** 월요일 아침 출근 지하철에서 슬랙 알림이 켜진다. 팀장이 노션 링크 5개를 보냈다. "이번 주에 다 검토해주세요." 평소 같으면 별표 누르고 잊어버렸겠지만, QLINK 공유 시트로 바로 저장한다.
>
> 저장하면서 **할 일 3개를 한꺼번에 등록**한다:
> - "수요일 9시 회의 전 1차 검토" (1회 알림)
> - "내용 요약 메모 작성" (목요일 18:00 알림)
> - "다음 회의에 가져갈 의견 정리" (금요일 14:00 알림)
>
> 하나의 링크가 **세 개의 액션 카드**로 분해되어 할일 탭에 줄지어 뜬다. 회사 데스크에서 PC로 같은 링크를 펼치면 같은 할 일들이 그대로 보이고, 하나씩 체크해 나간다.

#### 시나리오 2 — 취준생 채용 지원

> **박서연, 26세, 취업 준비 6개월 차.** 사람인·잡코리아·원티드에서 매주 10~15개 채용 공고를 찜한다. 그동안의 가장 큰 후회는 **"마감일 놓쳐서 지원조차 못 한 회사가 3~4곳 있었다는 것"**.
>
> QLINK로 채용 공고 링크를 저장하면서 자동으로 회사명과 마감일을 추출한다. 서연은 한 공고에 다음 할 일을 등록한다:
> - "자기소개서 초안 작성" (마감 3일 전 알림)
> - "포트폴리오 PDF 첨부 확인" (마감 1일 전 알림)
> - "지원 완료" (마감 당일 23:59 알림)
> - 면접이 잡히면 → "면접 D-day 매일 9시 복습" (반복 알림, 면접 날까지)
>
> 모든 채용 공고가 한 폴더에서 **마감일 순으로 자동 정렬**되고, 할일 탭에서 "기간 지남" 필터로 놓친 게 없는지 한눈에 확인.

#### 시나리오 3 — 시험·자격증 학습

> **이민호, 28세, 정보처리기사·SQLD 동시 준비 + IT 부트캠프 수강.** 인프런·유튜브 강의 링크가 50개를 넘었다. 매일 같은 시간에 진도를 나가고 싶지만, 캘린더 앱에 일정을 따로 만들면 강의 링크와 분리되어서 결국 안 본다.
>
> QLINK에서 강의 링크 하나에 **반복 할 일**을 등록한다:
> - "1편 듣기 — 매일 21:00" (반복: 평일, 일주일간)
> - "기출 풀어보기 — 매주 토요일 10:00" (반복: 매주)
> - "오답 노트 정리" (1회, 시험 1주일 전)
>
> 강의 링크 하나가 **30번의 알림**을 만들어내고, 전부 그 링크에 묶여 있다. 알림에서 바로 강의로 진입, 한 클릭에 다음 진도.

### 시장 기회

기존 북마크 / 링크 정리 서비스는 두 갈래로 나뉜다.

- (a) **PC·웹 클리퍼 중심** (Raindrop.io, Pocket): 데스크톱 전제, 컬렉션은 강하지만 행동 변환 약함
- (b) **자동 정리 AI** (mymind): 월 $7~13 유료, 이미지·시각 자료 중심, 액션 변환 없음

**그 어느 쪽도 "링크 → 할 일 → 반복 알림 → 완료" 흐름을 1차 기능으로 두지 않는다.** QR 스캔 + 모바일·웹 동시 PWA + AI 한 줄 요약 + **다중 할 일·반복 알림** + 공유 폴더 + 크롬 즐겨찾기 import를 한 자리에 묶은 한국어 친화 서비스는 비어 있다.

카톡 링크 공유 문화·QR 결제·자격증 강의 시청·다중 디바이스 사용·**채용 마감일·회의 D-day 같은 시간 압박** 같은 한국 사용자 행동에 맞춘 진입점이 차별점.

### 차별화 포인트 (요약)

1. **🎯 링크를 행동으로 전환** — 분석한 5개 중 "링크당 다중 할 일 + 반복 알림"을 1차 기능으로 둔 서비스 0개. **이게 QLINK의 메인 USP.**
2. **QR 스캔을 1차 입력으로 전면화** — 5개 중 카메라 QR을 메인에 둔 서비스 없음
3. **요약을 "읽기 전 필터"로 사용** — 저장 직전에 "이 링크 볼 가치 있나"를 판단
4. **외부 AI 계정 세션 활용 구조** — 운영 비용 0원 지향
5. **단일 PWA로 모바일·웹 동시 제공** — 설치 마찰 0, 한 계정 끊김 없는 동기화
6. **공유 폴더를 부수 기능 아닌 1차 기능으로** — 단톡방 링크 묻힘 문제 해결
7. **크롬 즐겨찾기 HTML import** — 통제 잃은 즐겨찾기 한 번에 정돈
